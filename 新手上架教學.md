# 新手上架教學

你不用會寫程式，只要照下面做。

## 你需要先有兩個帳號

1. GitHub 帳號  
   用來放網站檔案。

2. Render 帳號  
   用來把網站變成公開網址。

Render 比較適合這個網站，因為你的網站需要後端即時抓 BingX K 線，不是純靜態網頁。

## 第一步：把網站檔案上傳到 GitHub

1. 打開 GitHub。
2. 按右上角 `+`。
3. 選 `New repository`。
4. Repository name 填：

```text
crypto-ai-trader
```

5. 選 `Public` 或 `Private` 都可以。
6. 按 `Create repository`。
7. 進到新專案後，按 `uploading an existing file`。
8. 把這個資料夾裡的 `crypto_ai_trader_deploy.zip` 解壓縮。
9. 把解壓縮後的檔案全部拖到 GitHub 上傳。
10. 按 `Commit changes`。

## 第二步：到 Render 建立網站

1. 打開 Render。
2. 登入後按 `New +`。
3. 選 `Web Service`。
4. 選剛剛的 GitHub 專案 `crypto-ai-trader`。
5. 設定如下：

```text
Name: crypto-ai-trader
Runtime: Python
Build Command: pip install -r requirements.txt
Start Command: python local_app.py
Health Check Path: /api/status
```

6. Plan 選 Free。
7. 按 `Create Web Service`。

## 第三步：等 Render 部署完成

Render 會開始跑部署，第一次通常需要 2 到 5 分鐘。

完成後你會看到網址，長得像：

```text
https://crypto-ai-trader.onrender.com
```

打開這個網址，就是你的網站。

## 如果失敗

最常見是 GitHub 上傳錯層。

正確情況是 GitHub 專案最外層要直接看到這些檔案：

```text
main.py
local_app.py
requirements.txt
render.yaml
Procfile
web/
```

如果你看到的是：

```text
crypto_ai_trader/main.py
crypto_ai_trader/local_app.py
```

代表多包了一層資料夾，要重新上傳。

## 重要提醒

先不要放 BingX 下單 API Key 到公開網站。

目前網站只做掃描訊號，不會實盤下單，這樣比較安全。
